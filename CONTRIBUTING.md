## How to Contribute

### Reporting Issues

If you encounter a bug or have a feature request:

1. Check if the issue already exists in [Issues](https://github.com/wgtechlabs/container-build-flow-action/issues)
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Environment details (OS, GitHub Actions runner version, etc.)

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages (`git commit -m 'Add amazing feature'`)
6. Push to your fork (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Development Guidelines

- Keep scripts modular and well-documented
- Follow existing code style and conventions
- Test changes with actual workflows before submitting
- Update README.md if adding new features or changing behavior

### Testing

Before submitting:

- Verify the action works with all four flow types (pr, dev, patch, wip)
- Test with both Docker Hub and GHCR
- Ensure scripts handle edge cases gracefully

## Questions?

Feel free to open an issue for questions or reach out to the maintainers.
